00037 /* We would like to #include any system header file which could define
00038    iconv_t, 1. in order to eliminate the risk that the user gets compilation
00039    errors because some other system header file includes /usr/include/iconv.h
00040    which defines iconv_t or declares iconv after this file, 2. when compiling
00041    for LIBICONV_PLUG, we need the proper iconv_t type in order to produce
00042    binary compatible code.
00043    But gcc's #include_next is not portable. Thus, once libiconv's iconv.h
00044    has been installed in /usr/local/include, there is no way any more to
00045    include the original /usr/include/iconv.h. We simply have to get away
00046    without it.
00047    Ad 1. The risk that a system header file does
00048    #include "iconv.h"  or  #include_next "iconv.h"
00049    is small. They all do #include <iconv.h>.
00050    Ad 2. The iconv_t type is a pointer type in all cases I have seen. (It
00051    has to be a scalar type because (iconv_t)(-1) is a possible return value
00052    from iconv_open().) */
00053
